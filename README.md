# QA Functional Testing Project

## Role
Junior QA Engineer (Manual Testing)

## Project Overview
This project demonstrates hands-on manual functional testing on a modern web application featuring dynamic UI components.
The goal was to identify functional issues, validation gaps, UI inconsistencies, and error-handling problems through real user interaction scenarios.

## Scope of Testing
- Drag & Drop functionality
- Delayed and dynamic elements
- AJAX data loading
- Lazy loading images
- Infinite scroll
- Hidden elements
- Dynamic content generation
- File upload, processing & download
- iFrame and nested iFrame interactions
- Keyboard & mouse events
- Shadow DOM components

## Testing Type
- Manual Functional Testing
- UI & UX Validation
- Error Handling
- Performance Observations (non-automated)

## Tools Used
- Browser DevTools
- Jira (test cases & bug reporting)
- GitHub (documentation)

## Project Structure
- README.md – Project overview
- Test-Cases.md – Detailed functional test cases
- Bug-Reports.md – Identified bugs and observations
